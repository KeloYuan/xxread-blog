# 操作系统自举过程：先有鸡还是先有蛋？

> [!abstract] 核心概念
> 在计算机科学中，这个过程被称为"自举"（Bootstrapping）。程序员总是利用现有的、更强大的计算机系统，去开发下一个新系统。

## 历史案例

### Windows 1.0：在 DOS 上写出来（1985年）
- **开发环境**：MS-DOS（命令行界面）
- **开发语言**：C 语言 + 汇编语言
- **开发工具**：微软 C 编译器 + 宏汇编（MASM）
- **本质**：跑在 MS-DOS 之上的"图形壳子"

### 第一代 Mac OS：在 Lisa 上写出来（1984年）
- **开发环境**：Apple Lisa（比 Mac 更早、更贵、更高性能）
- **开发工具**：Lisa Workshop 开发环境
- **开发语言**：Pascal 语言 + 68000 汇编语言
- **技术**：交叉编译（在 Lisa 上写代码、编译，传给 Macintosh 测试）

## 从无到有的过程

### 最底层：物理开关
- 最早的计算机没有键盘和屏幕
- 程序员通过拨动面板上的物理开关，直接输入 0 和 1（机器码）

### 第一步：写出"汇编器"
- 用机器码写一个非常简单的程序，叫"汇编器"
- 不用记 0 和 1，而是记 MOV、ADD 这种助记符

### 第二步：写出"编译器"
- 用汇编语言写一个 C 语言编译器
- 可以写出更复杂的程序

### 第三步：写出"操作系统"
- 有了 C 语言编译器，可以高效编写大型软件
- Unix、Linux 以及后来的 Windows 内核大多这样诞生

## 层层递进的阶梯

就像人类文明：用石头打造石斧 → 用石斧砍木头做木屋 → 用木屋里的工具造出炼钢炉 → 造出飞机大炮

- **现在**：用 Mac/Windows 开发 鸿蒙、Android
- **40年前**：用小型机（DEC VAX）或 Lisa 开发 Windows 和 Mac OS
- **60年前**：用纸带、穿孔卡片或物理开关在大型机上写出第一个编译器

## 有趣的事实

现在的鸿蒙系统（尤其是底层内核）大量使用 C/C++ 编写，而这些 C 语言代码所依赖的编译器（如 LLVM/Clang），其最早的版本也是通过更早的系统"自举"进化而来的。
