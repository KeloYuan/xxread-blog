# Retina 与 HiDPI 显示技术详解

> [!abstract] 核心要点
> Retina 显示的本质是：高像素密度 + 合理缩放渲染，让正常观看距离下很难看到像素颗粒，文字边缘更细腻。

## 核心概念

### 三个"分辨率"概念（macOS）
1. **面板原生（物理像素 px）**：屏幕硬件真实像素数（固定）
2. **看起来像（逻辑点 pt）**：UI 排版用的逻辑坐标
3. **实际渲染（backing store px）**：系统实际把界面画到一张画布上的像素尺寸

### 理想路径（最锐利）
逻辑 pt × 2 = 面板原生 px（纯 2×整数映射，不需要二次缩放）

## MacBook Air M2 13.6" 实例

### 关键数字
- **面板原生**：2560 × 1664 px
- **默认逻辑**：1470 × 956 pt
- **有效倍率**：约 1.74×

### 渲染流程
1. 逻辑：1470×956 pt
2. 先 2×渲染：2940×1912 px
3. 再把 2940×1912 缩小到面板 2560×1664 显示
4. 缩放系数约：0.8707

### 最干净的 Retina 对齐档位
**1280 × 832（逻辑 pt）**
- 1280×2=2560，832×2=1664
- 渲染=面板原生，1:1 显示，理论上最"硬朗/锐利"

## 按钮渲染示例

如果按钮是 30×50 pt：
- **渲染（2×）**：30×50 pt → 60×100 px
- **默认档位**：60×100 px × 0.8707 ≈ 52×87 个物理像素（经过重采样）
- **最锐利档（1280×832）**：60×100 渲染像素会 1:1 显示到屏幕

## Windows 显示问题

### 常见"糊"的来源
1. 分数缩放（125%/150% 等）更容易出现取整/重采样带来的软边
2. 老应用不支持高 DPI → 被系统当"位图截图"整体拉伸
3. ClearType 子像素没调好/子像素顺序不匹配 → 字体彩边发虚
4. 低 PPI 屏 + 缩放 + 混合新旧应用

### Windows 缩放建议
- **推荐**：125% 或 150%（办公/编程舒服）
- **追求最"整数稳"**：200%（很清晰但工作区变小）
- **尽量别用**：自定义缩放

## 便携屏选择建议

### 低预算思路
- **最稳**：4K（3840×2160）+ 用 1920×1080 HiDPI
- **性价比路线**：
  - 2560×1600（16:10）：可用 1280×800 HiDPI
  - 2560×1440（16:9）：可用 1280×720 HiDPI
- **不推荐**：1920×1080 便携屏（清晰/空间两者很难兼得）
